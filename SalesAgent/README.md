# Sales Agent — внутренняя CRM для B2B-лидогенерации

Ищет и оценивает потенциальных клиентов для продажи growth/SEO/маркетинг-консалтинга (услуга —
то же, что уже сделано для ZHELEZO: технический аудит, SEO, блог, growth-стратегия), готовит
персонализированные черновики писем. **Ничего не отправляет автоматически** — только черновики
для ручной проверки и отправки.

Полная архитектура, решения и честные ограничения — в `docs/ARCHITECTURE.md` (обязательно
прочитать перед использованием — там объяснено, что реально автоматично, а что нет).

## Быстрый старт

```bash
cd SalesAgent
npm install
npm run init-db

# 1. Настроить своё имя/услугу/контакт
#    отредактировать src/messages/config.mjs

# 2. Собрать список сайтов-кандидатов (вручную, или попросить меня найти через веб-поиск)
#    seeds/example-seeds.json — формат-образец

# 3. Прогнать пайплайн: анализ сайта → поиск контактов → лид → скоринг
npm run import-seed -- seeds/ваш-файл.json

# 4. Посмотреть результат
npm run dashboard   # dashboard/index.html — открыть в браузере
npm run report -- weekly

# 5. Подготовить черновик письма для конкретного лида (id — из dashboard)
npm run draft -- 1 first_email
```

## Что это НЕ делает
- Не скрапит LinkedIn (нарушает их ToS).
- Не отправляет email/сообщения сама — только черновики.
- Не ищет новых кандидатов "с нуля по ключевым словам" полностью автономно — см.
  `docs/ARCHITECTURE.md`, раздел "Lead Discovery: что реально автоматично, а что нет".

## Компоненты
| Модуль | Что делает |
|---|---|
| `src/analyzer` | Company Analyzer — сайт, описание, соцсети, tech stack (только публичный сайт компании) |
| `src/contacts` | Contact Discovery — email/форма обратной связи с той же страницы, что нашёл analyzer |
| `src/scoring` | Lead Scoring — rule-based, веса настраиваются в `scoring/config.mjs` |
| `src/messages` | Message Generator — персонализированные черновики, честный "insight" на основе реальных находок |
| `src/crm` | Хранение: компании/контакты/лиды/история взаимодействий/черновики (SQLite) |
| `src/dashboard`, `src/reports` | Статичный HTML-дашборд + markdown-отчёты (daily/weekly/monthly) |
