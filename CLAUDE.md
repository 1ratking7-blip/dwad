# CLAUDE.md — регламент работы Claude Code в проекте CasinoReferral

Роль и цели проекта описаны в `GEMINI.md` (affiliate-маркетинг ZHELEZO / BC.Game) — этот файл
их не дублирует, а описывает конкретно рабочий процесс Claude Code в этом репозитории:
библиотеку экспертов в `/Experts` и правила их использования.

## Библиотека экспертов (`/Experts`)

В `/Experts/<Имя>/SYSTEM.md` — 26+ подробных ролевых инструкций (специализация, алгоритм
работы, чек-листы, типичные ошибки, лучшие практики, критерии качества). Это не отдельные
агенты/подпроцессы — это структурированная база знаний, которую нужно **прочитать и применить
как чек-лист** перед соответствующей работой, в рамках этой же сессии.

### Правило: сначала эксперт, потом работа

Перед выполнением задачи определи, какой раздел `/Experts` подходит, прочитай его
`SYSTEM.md` и следуй его чек-листам/алгоритму. Примеры маппинга (не исчерпывающий список —
ориентируйся по сути задачи, не только по ключевым словам):

| Задача | Экспert(ы) |
|---|---|
| Изменение кода (frontend/backend) | Backend/Frontend/FullStack + QA + Security |
| Изменение дизайна/UI | UX_UI + Designer |
| Продвижение сайта, SEO, контент | SEO + Marketing + Affiliate |
| Юридические вопросы, комплаенс | Lawyer |
| Деньги, монетизация, бюджет | Business + Finance |
| Анализ конкурентов, исследование рынка | Research + Analyst |
| Оптимизация скорости/инфраструктуры | Performance + DevOps |
| Автоматизация, скрипты, боты | Automation + AIEngineer |
| Копирайтинг, тексты | Copywriter (+ Marketing для стратегии) |
| SMM/соцсети | SMM + Copywriter |
| Продажи/лидген | Sales + Affiliate |
| Продуктовые решения (что делать, приоритеты) | ProductManager (+ ProjectManager для сроков) |
| Промпты для LLM/агентов | PromptEngineer |
| Data-анализ, метрики, статистика | DataScientist + Analyst |

### Пересекающиеся задачи — совместный вывод

Если задача затрагивает несколько областей, читай `SYSTEM.md` каждого релевантного эксперта и
явно сведи их рекомендации перед тем, как действовать — особенно если чек-листы одного эксперта
противоречат другому (например Marketing хочет агрессивнее, Lawyer — консервативнее: в
гемблинг-нише по умолчанию побеждает Lawyer/комплаенс, см. `GEMINI.md` п.8).

## Самообучение: новые эксперты

Если в процессе работы всплывает область, которой нет в `/Experts` — создай новую папку и
`SYSTEM.md` по той же структуре (см. любой существующий файл как образец полноты), и
используй её в дальнейшем в этой и последующих сессиях.

## Постоянное улучшение

Когда в рамках задачи находишь: новую типичную ошибку, лучшую практику, полезный чек-пункт,
которого нет в соответствующем `SYSTEM.md` — добавь его туда же (кратко, по существу, без
воды), а не только в `Память/` сессии. `Память/` — это журнал того, что произошло; `/Experts`
— это накопленная экспертиза, которая должна расти от сессии к сессии.

## Ограничение
Эксперты в `/Experts` — это письменные инструкции, а не самостоятельно думающие сущности.
Если задача требует реальной параллельной работы нескольких независимых потоков (не просто
чтения чек-листа) — используй обычный механизм подагентов (Agent tool), а не притворяйся, что
"эксперты сами обсуждают" где-то за кадром.
