# Security Report — ZHELEZO Landing

Дата: 2026-07-18. Объём: `Landing/` (React/TS/Vite, статический сайт на Cloudflare Pages,
без бэкенда/БД). Компилирует находки из двух предыдущих проходов
(`Landing/REPORT.md`, коммит `39e386b`) и текущей сессии — не дублирует детали, а даёт
итоговую картину с приоритетами.

## Найденные и исправленные проблемы

| # | Проблема | Серьёзность | Статус | Коммит |
|---|---|---|---|---|
| 1 | CSP использовал бы `unsafe-inline` (риск XSS через инлайн-скрипты) | Высокая | ✅ Исправлено — sha256-хэши вместо unsafe-inline | `39e386b` |
| 2 | Отсутствовал HSTS-заголовок | Средняя | ✅ Исправлено — `max-age=63072000; includeSubDomains; preload` | `377c754` |

| 4 | Cloudflare Pages отдавал 200+index.html на любой несуществующий путь (fingerprinting/SEO-мусор) | Низкая | ✅ Исправлено — `public/404.html` | `abcef4b` |
| 5 | React-дерево без ErrorBoundary — любое исключение при рендере роняет весь UI в белый экран, включая единственный путь к реферальной ссылке | Средняя (бизнес-риск на аффилиат-лендинге) | ✅ Исправлено — `ErrorBoundary` + репортинг ошибок в GA4/Metrica | `80c9579` (эта сессия) |

## Проверено и признано чистым (без изменений)

- **XSS/инъекции**: весь `src/**/*.tsx` — нет `dangerouslySetInnerHTML`, `eval`, `innerHTML`.
  Единственная динамическая вставка в DOM (`FAQ.tsx`, JSON-LD script) использует `.text =`
  (безопасное текстовое присваивание), не `innerHTML`, и корректно убирается в cleanup эффекта.
- **Секреты**: полный grep по репозиторию и git-истории (`git log --all --diff-filter=A`) —
  ни одного `.env`/credential-файла не коммитилось. GA4/Metrica ID — публичные идентификаторы,
  не секреты.
- **CSP-хэши в актуальном состоянии**: перегенерированы (`node scripts/csp-hashes.mjs`) после
  всех правок этой сессии и сверены построчно с `public/_headers` — совпадают, дрейфа нет.
- **CSRF/SSRF/SQLi/Command Injection/IDOR/Broken Auth/Session Fixation/JWT**: неприменимо —
  сайт полностью статический, нет сервера, БД, форм с бэкенд-обработкой, авторизации.
- **rel=noopener**: все `target="_blank"` (референс на BC.Game во всех CTA) имеют
  `rel="noopener noreferrer"`.
- **Cookie/CORS**: `Access-Control-Allow-Origin: *` — дефолт самого Cloudflare Pages для
  статики, не наша конфигурация; риска нет (нет cookie-сессий/приватных данных для утечки
  кросс-origin). Cookies ставит только сторонний скрипт Яндекс.Метрики — см. Best Practices
  ниже.
- **Dependency vulnerabilities**: `npm audit` (эта сессия) — **0 уязвимостей** (11 prod +
  127 dev + 32 optional зависимостей).
- **Supply chain**: зависимостей немного (framer-motion, lucide-react, react/react-dom,
  типовой Vite/Tailwind тулчейн) — все от известных мейнтейнеров, без подозрительных
  post-install скриптов.

## Оставшиеся риски

| Риск | Серьёзность | Почему не исправлено | Рекомендация |
|---|---|---|---|
| Apex-домен `zhelezo.space` (без www) не открывается по HTTPS (таймаут) | Средняя (UX/доверие, не эксплуатируемая уязвимость) | Нет доступа к Namecheap/Cloudflare DNS API в проекте (`.env`, `GEMINI.md`, `Память/context.md` проверены — токенов нет) | Перенести DNS-зону на Cloudflare и настроить redirect на уровне Cloudflare, либо включить PremiumDNS/SSL-форвардинг на Namecheap |
| Best Practices 77/100 (не 100) | Низкая | Полностью объясняется cookies стороннего скрипта Яндекс.Метрики (`mc.yandex.ru`) — вне нашего кода | Убрать можно только вместе с самой Метрикой — компромисс между аналитикой и баллом, решение бизнес-уровня, не техническое |
| Fabricated "live wins" в `SocialProof.tsx` (полностью симулированные данные, не реальные события) | Юридическая/репутационная, не техническая | Продуктовое решение, не код-баг — компонент уже сознательно оставлен предыдущей сессией как "реальный CRO-элемент" | См. `BUG_REPORT.md` — рекомендация: либо явно пометить как иллюстративные данные, либо заменить на реальную статистику, если/когда появится API BC.Game с реальными выигрышами |

## Рекомендации по приоритету

1. **Высокий приоритет**: получить доступ к DNS (Namecheap API-токен или перенос зоны на
   Cloudflare) — единственная незакрытая техническая проблема с реальным пользовательским
   воздействием (часть посетителей, набравших голый домен, не смогут открыть сайт).
2. **Средний приоритет**: решить, что делать с `SocialProof.tsx` — юридический/бизнес-риск,
   не техдолг (см. `BUG_REPORT.md`).
3. **Низкий приоритет**: Best Practices 77 — осознанный компромисс ради аналитики, не требует
   действий, если устраивает текущий трейд-офф.
