# ZHELEZO Discord bot

Реализация бота из `Texts/discord_server_structure.md`: `!spin`, `!bonus`, `!faq <тема>`,
`!leaderboard`. Код готов к запуску — не хватает только токена бота (см. ниже), который я не
могу получить сам: это требует входа в ваш Discord-аккаунт и создания приложения на
Developer Portal.

## Что нужно сделать перед первым запуском

1. **Создать приложение бота**: https://discord.com/developers/applications → New
   Application → Bot → скопировать токен ("Reset Token" если создаёте впервые).
2. **Включить привилегированный интент**: там же, в разделе Bot → "Message Content Intent" —
   включить (без этого бот не увидит текст команд `!spin` и т.п.).
3. **Пригласить бота на сервер**: OAuth2 → URL Generator → scope `bot` → права (permissions):
   Send Messages, Read Message History, Read Messages/View Channels — вставить сгенерированную
   ссылку в браузер и выбрать ваш сервер ZHELEZO.
4. **Установить зависимости и токен**:
   ```bash
   cd Scripts/discord-bot
   npm install
   ```
   Задать токен через переменную окружения (не хардкодить в файл, не коммитить):
   ```bash
   DISCORD_BOT_TOKEN=ваш_токен npm start
   ```
   Или создать `.env` рядом (уже в `.gitignore` на уровне репозитория) и подключить через
   `dotenv`, если предпочитаете — сейчас код читает `process.env.DISCORD_BOT_TOKEN` напрямую.

## Команды

| Команда | Что делает |
|---|---|
| `!spin` | Отмечает участника в дневном/недельном леборде (1 раз в день, файл `data/spins.json`) |
| `!bonus` | Присылает реф-ссылку в личные сообщения (не в чат — см. `discord_server_structure.md`) |
| `!faq <тема>` | Отвечает по одной из тем: `fair`, `kyc`, `withdraw`, `bonus` (см. `faq.js`) |
| `!leaderboard` | Топ-10 по спинам за текущую неделю |

## Хранилище
`data/spins.json` — простой JSON-файл (в `.gitignore`, не коммитится). Этого достаточно для
одного сервера на разумном масштабе. Если участников станет много тысяч и файл станет узким
местом — тогда стоит переходить на настоящую БД, не раньше.

## Что не сделано
Деплой/хостинг самого процесса (нужен сервер/PaaS, где бот будет работать постоянно — Railway,
Fly.io, обычный VPS и т.п.) — отдельное решение по инфраструктуре, не входит в код.
