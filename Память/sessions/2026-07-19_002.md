## Дополнение 2 (в ту же сессию): построена автоматическая SMTP-отправка, пользователь
подтвердил и предоставил реальные креды — письма реально отправлены

Пользователь явно подтвердил ("Да, отправить все 13 автоматически") после того, как я
переспросил с прямым описанием риска (гемблинг-ниша, 13 получателей, без ручной проверки
каждого). Собрано:
- `src/messages/mailer.mjs` — тонкая обёртка над `nodemailer`, кидает исключение при ошибке,
  сама ничего не помечает отправленным.
- `bin/send.mjs` (`npm run send`) — по умолчанию dry-run, реальная отправка только с флагом
  `--confirm`, `--limit`/`--delay-ms` (по умолчанию 8с между письмами), лиды без найденного
  email пропускаются (только tentenseven.com — у него лишь контактная форма).
- `markDraftSent()` в `crm/index.mjs` теперь принимает опциональный `note` — чтобы отличать
  автоматическую отправку от ручной пометки; обновлён устаревший doc-комментарий (раньше
  утверждал, что в коде нет вызовов SMTP — это больше не так по архитектуре).
- `.env.example` — креды через `SMTP_*`, используется нативный `--env-file-if-exists`
  (Node 22+, без лишней зависимости `dotenv`). `.env` подтверждён как gitignored
  (`git check-ignore`) — реальный пароль в git никогда не попадал.
