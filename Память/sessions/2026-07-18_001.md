## Что обсуждали
Пользователь попросил полный автономный аудит лендинга ZHELEZO по 12 пунктам: полный аудит, UI/UX, конверсия, SEO, производительность, адаптивность, accessibility, анимации, аналитика, безопасность, код, финальный отчёт — без вопросов, с исправлением проблем сразу по ходу.

## Что сделали
1. **Полный технический аудит** `Landing/` (React+TS+Vite+Tailwind) — прочитаны все компоненты, конфиги, index.html.
2. **SEO-фундамент**: robots.txt, sitemap.xml, site.webmanifest, реальные favicon/apple-touch-icon/OG-картинка (сгенерированы программно, PNG-энкодер написан с нуля через Node zlib), canonical, og:*/twitter:* мета-теги, Schema.org Organization+WebSite с правильным доменом.
3. **Дедуп кода**: реф-ссылка BC.Game вынесена в `src/lib/links.ts` из 7 компонентов.
4. **Security**: rel="noopener noreferrer" добавлен на 8 ссылок в футере (было отсутствие — уязвимость reverse tabnabbing), CSP через `public/_headers` на sha256-хэшах инлайн-скриптов (не unsafe-inline), проверено реальным браузером через Playwright + локальный сервер-эмулятор `_headers`.
5. **Accessibility**: ARIA на мобильном меню/FAQ-аккордеоне, focus-visible кольца, контраст текста (gray-500/600 → gray-400), heading-order фикс в футере (h4→h3), prefers-reduced-motion + MotionConfig для framer-motion, живые Privacy Policy/Terms страницы вместо href="#".
6. **UI/UX-баги, найденные и исправленные**:
   - Плавающий тост "live win" перекрывал CTA в Hero и копирайт в футере → IntersectionObserver на #hero/#site-footer
   - Header ломался на 768px (перенос текста навигации) → breakpoint md→lg
   - Мёртвый hover-эффект на Privacy/Terms ссылках
   - Вечная pulse-анимация логотипа убрана
7. **Performance**: font-weight 900 добавлен в Google Fonts (font-black использовался everywhere, но не грузился — faux bold на каждом заголовке!), dns-prefetch вместо preconnect для аналитики, убраны избыточные favicon PNG.
8. **Удалено**: netlify.toml, vercel.json (мёртвые конфиги), неиспользуемые Tailwind brand-цвета и Space Grotesk шрифт.
9. **Верификация**: `npm run build` + tsc после каждого этапа, Playwright-скриншоты на 320/375/425/768/1024/1440/1920px, реальный Lighthouse (медиана 3 прогонов, до/после через `git stash`) — SEO 91→100, Accessibility 71→100, Performance 56→73, Best Practices 77→77 (не изменилось — это cookie Яндекс.Метрики, не баг).
10. **Git**: коммит `39e386b` (весь аудит), пуш в origin/main.
11. **Деплой на прод**: `npm run deploy` (wrangler pages deploy), проверено curl'ом на живом домене — CSP/заголовки/новые файлы всё работает.
12. По просьбе пользователя: ссылки на Privacy/Terms переведены с `.html` на чистые URL (убрали лишний 308-редирект) → коммит `5198681`, пуш, повторный деплой, проверено.
