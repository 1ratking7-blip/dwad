## Важная находка: деплой НЕ автоматический
В процессе проверки, долетели ли изменения (код Яндекса, вставленный несколькими часами ранее
в этой же расширенной сессии) до живого сайта — обнаружил, что **нет**. Все изменения тонны
исправно коммитились и пушились в `main`, но сайт хостится на Cloudflare Pages **без
GitHub-интеграции auto-deploy** — деплой ручной, через `npm run deploy` (`wrangler pages deploy
dist --project-name=zhelezo`), см. `Landing/README.md`. Ранее в этой же расширенной сессии я
ошибочно писал пользователю "Cloudflare Pages подхватит и задеплоит автоматически" — это было
неверное предположение, не проверенное на практике. Только сейчас, спустя часы, реально
задеплоил все накопленные изменения (Yandex-код + BreadcrumbList + новая статья) одной
командой — подтверждено `curl` на живом домене после деплоя, все три правки видны.

**Вывод на будущее (важно для всех следующих сессий с изменениями Landing/):** после любого
коммита, затрагивающего `Landing/`, обязательно запускать `npm run deploy` из `Landing/`, иначе
изменения останутся только в git и не попадут на прод. Стоит явно проверять это в конце каждой
сессии, а не полагаться на предположение об автодеплое.
