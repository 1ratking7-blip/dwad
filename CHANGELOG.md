# CHANGELOG — ZHELEZO (zhelezo.space)

Реальные изменения по датам, в обратном хронологическом порядке. Не журнал сессий (см.
`Память/sessions/*.md` для полного контекста) — только то, что фактически изменилось в
продукте/коде. Заведён 2026-07-21 по запросу пользователя, когда main-поток принял на себя
постоянную роль CTO/Growth Engineer для проекта.

## 2026-07-21

- Создан этот файл и передана роль постоянного технического владельца проекта от фонового
  форка к основному потоку (по явному запросу пользователя) — во избежание повторного
  конфликта одновременного редактирования одних и тех же файлов (см. инцидент с
  `SalesAgent/bin/send.mjs` тем же днём ранее).
- **Fix**: добавлена skip-to-content ссылка (RU/EN/VI) — раньше клавиатурный/скринридер-
  пользователь был вынужден таб-ать через весь header на каждой странице.
- **Fix**: LanguageSwitcher теперь закрывается по Escape с возвратом фокуса на кнопку.
- **Chore**: `Scripts/brand-monitor/run-checks.bat` — обёртка для периодического запуска
  site-health-check.mjs + check-mentions.mjs, протестирована end-to-end.
- **Automation**: зарегистрирована задача Windows Task Scheduler `ZHELEZO-BrandMonitor` —
  ежедневный запуск `run-checks.bat` в 09:00, лог в `logs/monitor.log`. Подтверждено
  явным разрешением пользователя (системное изменение вне репозитория), проверено
  `schtasks /query` — статус Ready.
- **Fix (a11y)**: все 17 внешних ссылок (`target="_blank"`) на сайте теперь объявляют
  скринридеру "открывается в новом окне" (visually-hidden span, переведён на RU/EN/VI).
  Декоративные иконки внутри этих ссылок получили `aria-hidden="true"`, чтобы не дублировать
  объявление. Соотношение 17 ссылок / 17 меток проверено `grep` перед коммитом. Задеплоено,
  подтверждено `curl` против реального JS-бандла (`LocaleContext-*.js`) — по одному
  вхождению строки на каждый из 3 языков.
