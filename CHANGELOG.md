# CHANGELOG — ZHELEZO (zhelezo.space)

Реальные изменения по датам, в обратном хронологическом порядке. Не журнал сессий (см.
`Память/sessions/*.md` для полного контекста) — только то, что фактически изменилось в
продукте/коде. Заведён 2026-07-21 по запросу пользователя, когда main-поток принял на себя
постоянную роль CTO/Growth Engineer для проекта.

## 2026-07-21

- Создан этот файл и передана роль постоянного технического владельца проекта от фонового
  форка к основному потоку (по явному запросу пользователя) — во избежание повторного
  конфликта одновременного редактирования одних и тех же файлов (см. инцидент с
  `SalesAgent/bin/send.mjs` тем же днём ранее).
- **Fix**: добавлена skip-to-content ссылка (RU/EN/VI) — раньше клавиатурный/скринридер-
  пользователь был вынужден таб-ать через весь header на каждой странице.
- **Fix**: LanguageSwitcher теперь закрывается по Escape с возвратом фокуса на кнопку.
- **Chore**: `Scripts/brand-monitor/run-checks.bat` — обёртка для периодического запуска
  site-health-check.mjs + check-mentions.mjs, протестирована end-to-end.
- **Automation**: зарегистрирована задача Windows Task Scheduler `ZHELEZO-BrandMonitor` —
  ежедневный запуск `run-checks.bat` в 09:00, лог в `logs/monitor.log`. Подтверждено
  явным разрешением пользователя (системное изменение вне репозитория), проверено
  `schtasks /query` — статус Ready.
- **Fix (a11y)**: все 17 внешних ссылок (`target="_blank"`) на сайте теперь объявляют
  скринридеру "открывается в новом окне" (visually-hidden span, переведён на RU/EN/VI).
  Декоративные иконки внутри этих ссылок получили `aria-hidden="true"`, чтобы не дублировать
  объявление. Соотношение 17 ссылок / 17 меток проверено `grep` перед коммитом. Задеплоено,
  подтверждено `curl` против реального JS-бандла (`LocaleContext-*.js`) — по одному
  вхождению строки на каждый из 3 языков.
- **Fix**: `public/404.html` был русскоязычным независимо от того, в какой языковой секции
  (`/en/`, `/vi/`) пользователь словил битую ссылку — Cloudflare Pages отдаёт один и тот же
  статический 404-файл на все несовпавшие пути, отдельного 404 на директорию в статическом
  хостинге не бывает. Добавлен небольшой inline-скрипт без зависимостей, который определяет
  языковой префикс по URL и подменяет заголовок/текст/ссылку "на главную" на EN/VI; RU
  (без префикса) не тронут. Задеплоено, HTTP-код 404 сохранён, наличие скрипта подтверждено
  `curl` для `/en/` и `/vi/`.
- **Audited, ничего не меняли (честно, не для галочки)**: изображения (все статичные ассеты
  ≤56 КБ, оптимизировать нечего), шрифты (уже preconnect+preload+`display=swap`+noscript-
  фоллбэк — best practice), состав JS-бандла (крупнейший чанк 266 КБ / 86 КБ gzip — разумно
  для React-приложения, ничего раздутого), canonical-теги (все 24 HTML-страницы блога +
  3 языковые версии главной корректно self-referencing, проверено построчно), og:image:alt
  ("ZHELEZO Crypto Gaming" одинаков на всех языках — это имя бренда, не забытый перевод),
  формы на сайте отсутствуют (нечего валидировать), SalesAgent (`npm audit` чисто, синтаксис
  чист, TODO/FIXME не найдено — тех.долга не накопилось).
- **Security fix**: `SalesAgent/src/reports/index.mjs`'s `generateStats()` строил SQL через
  сырую интерполяцию строки (`WHERE created_at >= '${since}'`) вместо параметризованных
  запросов, как во всём остальном коде. Не эксплуатировался через текущие вызовы (оба
  передают только внутренние/белый-список значения), но был реальной дырой для любого
  будущего вызова с пользовательским `since`. Переписано на параметризованные `?`-запросы;
  проверено на реальном payload'е `OR '1'='1'` — фильтр больше не обходится. Коммит `586e2d0`.
- **Audited, ничего не меняли**: заголовки безопасности (`_headers` — CSP на точных хэшах,
  HSTS, X-Frame-Options, Referrer-Policy, Permissions-Policy — уже полный набор), git-история
  на секреты (полный `git log --all -p` по api-key/secret/password/token — только плейсхолдер
  в `.env.example`, `.env` никогда не коммитился), CRO дальше Hero.tsx (9+ CTA-точек по
  странице, `Header` уже `fixed top-0` — мобильный доступ к CTA уже решён), мёртвый код
  Landing (`noUnusedLocals`/`noUnusedParameters` в tsconfig + Tailwind JIT структурно не
  дают ему накапливаться).
